<h2><a href="https://www.geeksforgeeks.org/problems/account-merge/1?page=1&difficulty=Hard&status=unsolved&sortBy=submissions">Account Merge</a></h2><h3>Difficulty Level : Difficulty: Hard</h3><hr><div class="problems_problem_content__Xm_eO"><p><span style="font-size: 18px;">You are given a 2D array <strong>arr[][]</strong>, of size <strong>n</strong>, where each element <strong>arr[i] </strong>is an array of strings representing an account, where the first element <strong>arr[i][0]</strong> is a name, and the rest of the elements are emails associated with this name. Two accounts belong to the same person if there is a <strong>common email</strong> associated with both accounts. A person can have any number of accounts initially, but all have the same name.</span></p>
<p><span style="font-size: 18px;">Your task is to <strong>merge</strong> these accounts and return them in the following format: the first element of each account is the <strong>name</strong>, and the rest of the elements are <strong>emails </strong>in<strong> lexicographically sorted</strong> order.</span></p>
<p><span style="font-size: 18px;"><strong>Note:</strong> Accounts themselves can be returned in any order. Even if two accounts have the same name, they may belong to different people as different people could have the same name.</span></p>
<p><span style="font-size: 18px;"><strong>Examples:</strong></span></p>
